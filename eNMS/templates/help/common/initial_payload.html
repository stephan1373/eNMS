The Initial Payload parameter provides starting data that can be referenced later by service(s). This parameter is a dictionary with key/value pairs. This parameter is always present and will default to "{ }".

Important note: only one Initial Payload matters during a run: the one populated for the service or workflow being run. If the user runs a workflow, the Initial Payload defined on the workflow will be used (and any service Initial Payload is ignored). If the user runs an individual service, that service's Initial Payload will be used if defined (and the workflow Initial Payload is ignored if defined).
